# Collection Method

## Goal

Measure where World Cup 2026 starting-lineup players were trained between ages 5 and 16.

## Core unit of analysis

The main unit is a `player-match` record:

- one player
- one national team
- one World Cup match
- one starting-XI appearance

Youth development is then attached to that player as one or more records spanning ages 5-16.

## Source hierarchy

Use sources in this order:

1. `FIFA.com`
   - Official list of qualified teams
   - Official match reports and match-centre pages
2. `Wikipedia`
   - Accessible backup for completed match lineups
   - Good for cross-checking starters, positions, substitutions, and match metadata
   - Also useful for a first-pass pull of player `Youth career` infobox data
3. `Transfermarkt`
   - Best widely available source for youth-club histories
4. `Official club`, `official federation`, and `reputable player biography` pages
   - Use when Transfermarkt is incomplete, missing, or ambiguous

## Dataset files

### `data/qualified_teams.csv`

One row per qualified team.

Fields: `team_name`, `fifa_code`, `confederation`, `qualification_status`, `notes`

### `data/world_cup_match_lineups.csv`

One row per starting player per match.

Required fields:

- `match_id`
- `match_date`
- `stage`
- `team_name`
- `opponent_name`
- `player_name`
- `player_wikipedia_url`
- `player_position`
- `starter`
- `source_primary`
- `source_secondary`
- `source_notes`

### `data/player_youth_training.csv`

One row per youth-club spell per player.

Required fields:

- `player_name`
- `national_team`
- `club_name`
- `club_wikipedia_url`
- `club_country`
- `start_age_est`
- `end_age_est`
- `start_year_est`
- `end_year_est`
- `source_primary`
- `source_secondary`
- `confidence`
- `notes`

### `data/training_club_summary.csv`

One row per national-team × youth-club combination. Generated by `scripts/summarize_training_clubs.py`.

Fields: `national_team`, `club_name`, `estimated_training_years`, `players_trained`

### `data/training_country_summary.csv`

One row per national-team × training-country combination. Generated by `scripts/summarize_training_countries.py`.

Fields: `national_team`, `club_country`, `estimated_training_years`, `players_trained`

## Age attribution rules

Because public sources usually give year ranges rather than exact birthdays plus exact registration dates, ages 5-16 often need estimation.

Use these rules consistently:

1. If a source gives exact youth years, record them directly.
2. If a source gives a span like `2008-2013`, convert that to estimated ages using the player's birth year.
3. If a spell overlaps the 5-16 window only partially, keep only the overlapping portion.
4. If the player began organized training after age 5, start at the earliest verifiable age.
5. If the source only lists a club with no years, leave the year fields blank and mark confidence as `low`.
6. If the source has a youth section but no row can be confidently clipped to ages 5-16, keep the earliest listed youth club as a `low` confidence fallback and leave the age/year fields blank.

## Confidence scale

- `high`: exact years or exact age bands from a reliable source
- `medium`: reasonable estimate from partial years or multiple corroborating sources
- `low`: club identified but timing is inferred or incomplete

## Quality checks

Before treating a player record as complete, verify:

1. The player actually started the World Cup match.
2. Youth clubs are ordered chronologically.
3. Estimated ages do not go backward or overlap incorrectly.
4. The 5-16 window is fully covered, or documented as partially missing.
5. Country names are standardized for later aggregation.

## Practical collection order

Work in this order for speed:

1. Populate all completed World Cup match starters.
2. Build a deduplicated player list from those starters.
3. Research youth-club histories for the most repeated starters first.
4. Backfill less-used starters afterward.

## What can be fully automated vs semi-manual

### Reasonably automatable

- qualified teams
- match metadata
- many starting-XI records
- player deduplication
- first-pass youth-club extraction from player Wikipedia pages
- club-country backfill from linked youth-club pages
- country standardization

### Likely semi-manual

- youth-club year verification
- early childhood clubs not consistently published
- resolving players with incomplete or conflicting biographies
